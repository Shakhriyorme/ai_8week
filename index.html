4 Production Portfolio Projects

Project #1 — Week 1-2
AIAS 🎓
Artificial Intelligence Attendance System — Multi-Object Tracking & Person Re-Identification
DEADLINE: Day 14

A production-grade smart attendance system that eliminates manual roll-calls entirely. Unlike Face-ID (which fails when students turn away or wear masks), AIAS uses body-based Re-Identification — tracking each person via their unique "digital silhouette" (body shape, clothing color, posture). An ESP32-CAM streams wirelessly to a YOLOv8 Nano detection engine; ByteTrack assigns persistent cross-frame IDs; OSNet extracts discriminative Re-ID embeddings. Zero frontal-camera requirement. Attendance logged to CSV/Excel automatically.

Visual InputESP32-CAM Wireless Stream
Tracking LogicByteTrack Association Metric
Re-ID EngineOSNet Omni-Scale Features
OutputAuto CSV / Excel Logging
YOLOv8 NanoByteTrackOSNet ESP32-CAMOpenCVFastAPI StreamlitSQLiteDocker
  • YOLOv8 Nano detector: real-time person detection optimized for edge — <25ms inference, no GPU required
  • ByteTrack MOT: persistent student IDs across full session — survives occlusion, crowd crossings, re-entry after leaving frame
  • OSNet Re-ID: omni-scale feature extraction from body silhouette + clothing — no frontal face needed, robust to masks & glasses
  • ESP32-CAM integration: wireless MJPEG stream over Wi-Fi → OpenCV RTSP capture → YOLOv8 pipeline (zero-cable classroom setup)
  • Auto attendance log: first-seen / last-seen timestamps per student ID → SQLite → daily CSV / Excel export
  • Streamlit dashboard: live annotated feed, real-time attendance table, absence alerts, per-student entry/exit timeline
📢 Presentation Script (10 min)

"Standard Face-ID has a fundamental flaw — it only works when a student looks directly at the camera. Put on a mask, turn your head, or sit in a crowded row and the system loses you. AttendIQ solves this with person Re-Identification: we encode each student as a 'digital silhouette' using OSNet's omni-scale feature pyramid, then ByteTrack maintains a persistent ID gallery frame-to-frame. The hardest challenge was gallery drift — when two students wear similar colored clothing. I solved it by fusing deep Re-ID embeddings with IoU-based spatial consistency. In future: cross-camera Re-ID across multiple ESP32-CAMs placed at different room angles." — Live demo: student enters, turns away, gets briefly occluded, re-appears — same ID maintained throughout.

AI-Powered Learning Prompts

Golden Rule: Never copy-paste AI code. AI is your tutor and questioner. Every prompt forces understanding. You write the code — AI explains the logic.

UnderstandLearning a New Concept

Use BEFORE studying a new topic:

You are an experienced AI/ML teacher. I want to learn [TOPIC]. 1. Assume I know nothing — start with core intuition, not formulas 2. Give one simple real-life analogy 3. Then go technical — but explain WHY at each step 4. End with 3 questions to test my understanding 5. If I answer wrong, guide me (don't just tell me) [TOPIC] = [e.g.: attention mechanism, Docker layers, gradient descent, YOLO architecture]
AnalyzeUnderstanding Code Logic

When reading existing code — understand, don't copy:

I'm reading this code. WITHOUT giving me the code itself, explain: 1. What does this code do? (1-2 sentences) 2. What algorithmic ideas are used? 3. If I wrote this from scratch, what steps would I follow? 4. What should I watch out for when implementing? [CODE DESCRIPTION — not full code, just describe what it does] Example: "This is a PyTorch custom Dataset class with __len__ and __getitem__"
TestSelf-Knowledge Check

After learning — test your understanding:

I learned [TOPIC]. Test me: 1. First I'll explain — you listen: [MY EXPLANATION] 2. Where am I wrong or incomplete? 3. What important part did I miss? 4. As a senior ML engineer, what interview question would you ask? 5. What answer would create a "wow, this person gets it" impression? [TOPIC] = PyTorch autograd / YOLO architecture / RAG chunking / Vector DB HNSW...
DebugUnderstanding Errors (Not Getting Fixes)

When code errors — DON'T ask for the fix directly:

My code gave this error: [ERROR MESSAGE] My code was trying to: [GOAL] WITHOUT giving me the fix directly: 1. What does this error mean? (plain English) 2. What causes this error? (2-3 possibilities) 3. How would I debug this step by step? 4. How do I prevent similar errors in the future?
ArchitectureDesigning Project Structure

Before starting a new project:

I want to build [PROJECT NAME]. Goal: [short description] Stack: [technology list] Give me: 1. Core system components (boxes + responsibilities) 2. Data flow: input → ... → output 3. Top 3 hardest technical challenges 4. Where would a senior engineer start first? 5. Minimum feature set for a 1-week MVP? NO CODE — architecture and ideas only.
InterviewPresentation & Interview Prep

Before a demo or interview:

I'm giving a 10-15 min technical presentation on [PROJECT/TOPIC]. Project: [short description] Audience: [teacher/employer/peers] 1. What questions might they ask? (Top 10 technical) 2. Which questions have "traps"? (dangerous to answer without thinking) 3. What opening sentence creates the most impact? 4. What weakness might the audience notice? 5. What should I say for a "this engineer really knows it" impression?
